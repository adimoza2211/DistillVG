# DistillVG: High-Performance Grounding Pipeline

DistillVG is a knowledge distillation pipeline designed to train a high-performance, mobile-ready Visual Grounding student model. The current state of the pipeline strictly uses production-ready, state-of-the-art encoders and eliminates all mock components, fallbacks, and dummy architectures.

## Current Pipeline Architecture

The pipeline consists of the following tightly integrated components:

### 1. Visual Backbone (YOLO FPN + RoI-Align)
- **Role:** Extracts localized visual features for grounding candidate regions.
- **Component:** Uses an Ultralytics YOLO26 frozen feature extractor.
- **Mechanic:** Extracts P3, P4, and P5 Feature Pyramid Network (FPN) feature maps from the input image. For every proposal generated by the YOLO head, `torchvision.ops.roi_align` is applied to these feature maps to spatially pool a feature vector (`roi_tokens`). A trainable linear projection layer brings these visual features down to the student's `hidden_dim`.
- **Note:** The underlying YOLO backbone is frozen during standard distillation to maintain its general-purpose feature representations, but the projection head is actively trained.

### 2. Text Encoder (MobileCLIP-S1)
- **Role:** Extracts semantic representations of the textual grounding query.
- **Component:** Apple's MobileCLIP-S1 Text Transformer.
- **Mechanic:** The raw text sequence is tokenized using the native CLIP BPE tokenizer, supporting a maximum context length of 77 tokens. The tokens are passed through the frozen MobileCLIP transformer layers. Instead of pooling to a single embedding, the pipeline extracts the full sequence of contextual embeddings (`[Batch, 77, 512]`). A trainable projection layer maps these 512-dimensional tokens to the student's `hidden_dim`.

### 3. Fusion Transformer
- **Role:** Cross-modal reasoning between text and visual proposals.
- **Component:** Multi-layer Transformer Encoder.
- **Mechanic:** Concatenates the text sequence tokens and the visual `roi_tokens` into a single sequence. Self-attention enables the model to perform deep, dense interactions between words in the query and specific visual regions in the image.

### 4. Detection / Grounding Heads
- **Role:** Final predictions for each proposal.
- **Components:**
  - **`bbox_head`**: Regresses tight bounding box coordinates (xyxy) from the fused object tokens.
  - **`verifier` head**: Classifies whether a visual proposal is the correct target for the given text query.

### 5. Proposal Generation (YOLO26)
- **Role:** Generates high-recall object proposals without textual knowledge.
- **Component:** Pretrained YOLO object detector.
- **Mechanic:** Evaluates the image dynamically at runtime and yields top bounding box proposals based on objectness/confidence thresholds (`yolo26_conf_threshold`, `yolo26_iou_threshold`). These act as the visual candidate anchors for the RoI-Align feature extractor.

### 6. Online Verifier (InternVL Teacher)
- **Role:** Acts as the "Teacher" to guide the student during distillation.
- **Component:** InternVL Large Vision-Language Model.
- **Mechanic:** Employs a rigorous Two-Stage validation process to score YOLO proposals against the text query.
  - **Stage 1 (Coarse):** Computes similarity between the base query and cropped proposal images.
  - **Stage 2 (Fine-grained):** Draws a red highlight box over the top-k proposals on the original image and asks the VLM a 4-class question to definitively score the match.
  - The verifier generates soft-targets which are heavily utilized by the DWBD and Consistency loss functions during the Phase 1 training step.

## Training Subsystems

The model trains across two primary phases:

*   **Phase 1 (Distillation):** YOLO generates candidate proposals. The heavy InternVL online verifier computes soft target scores. The MobileCLIP configures the textual embedding. The student learns to match the InternVL teacher's output distribution using DWBD (Dynamic Weight Box Distillation), KL-Divergence on verifier scores, and structural consistency losses.
*   **Phase 2 (Strict Fine-tuning):** The verifier is completely unloaded from GPU memory. The model focuses entirely on strict Box supervision and Ground Truth matches, fine-tuning the remaining layers to maximize absolute grounding accuracy.

## Running the Pipeline

**Critical execution rule:** Every command must be executed within the `distill` conda environment. Always use `conda activate distill`. 

### Quick Smoke Test (Phase 1)
```bash
conda activate distill && python scripts/train.py training.epochs=1 training.max_train_records=8 training.batch_size_per_gpu=2 data.num_workers=0 data.persistent_workers=false train.mode=phase1
```

### Full Phase 1 Training Run
```bash
conda activate distill && python scripts/train.py
```

### Phase 2 Strict Fine-Tuning Run
```bash
conda activate distill && python scripts/train.py train.mode=phase2_strict
```
